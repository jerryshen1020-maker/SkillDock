# SkillDock v1.1 需求总结

## 项目概述

| 项目 | 内容 |
|------|------|
| 名称 | SkillDock |
| 版本 | v1.1 |
| 类型 | macOS 独立应用 |
| 设计风格 | 现代浅色 SwiftUI |

## 背景与目标

### 痛点（来自 v1.0 反馈）
1. 路径需要手动输入，不方便
2. 不能递归扫描子文件夹内的 skill
3. 缺少默认全局目录
4. UI 与设计稿差异大，侧边栏太复杂
5. Skill 开关无效（未同步到 Claude 配置）

### 目标
- UI 改版，对齐设计稿
- 目录选择器（系统弹窗）
- 递归扫描子文件夹
- 默认导入 ~/.claude/skills/
- Skill 开关真正同步到 Claude 配置

## 功能范围

### 做什么
- **UI 改版**：入口式侧边栏 + 现代浅色配色
- **目录选择器**：使用 NSOpenPanel 弹出系统目录选择窗口
- **递归扫描**：自动递归遍历子目录，发现所有包含 SKILL.md 的文件夹
- **默认全局目录**：启动时自动导入 ~/.claude/skills/
- **Skill 开关生效**：修改 ~/.claude.json 的 allowedTools 实现真正限制

### 不做什么
- 不做 skill 的增删改
- 不做 skill 导入/导出
- 不接入 Trae/Trae CN 自动配置

## 关键技术约束

### Claude CLI 项目级配置结构
```json
// 项目目录: <project>/.claude/settings.json
{
  "permissions": {
    "allow": ["Skill(brainstorming)", "Skill(design-exploration)"],
    "deny": ["Skill(browser-use)"]
  }
}
```

### Skill 启用机制
- SkillDock 维护 .skills-config.json（本地存储，显示用）
- 真正生效：修改项目级 `<project>/.claude/settings.json` 的 permissions
- 优先级：Project > User，项目级配置覆盖全局设置

### 项目隔离原理
即使所有 skill 安装在全局目录 `~/.claude/skills/`，通过项目级 permissions 配置，可以实现：
- 项目 A：允许 brainstorming, design-exploration
- 项目 B：允许 prd-doc-writer, writing-assistant

### 目录结构
- 默认来源：~/.claude/skills/
- 支持最多 10 个来源目录
- 递归扫描子文件夹

## 关键决策

1. **入口式侧边栏**：简化侧边栏，点击入口切换内容区
2. **来源筛选整合**：在 Skills 仓库页面顶部添加来源筛选器
3. **系统目录选择器**：使用 NSOpenPanel 替代手动输入
4. **Skill 开关生效**：修改项目级 `<project>/.claude/settings.json` 的 permissions 配置
5. **项目隔离**：通过 Project 级配置实现不同项目使用不同 skill
