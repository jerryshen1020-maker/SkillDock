2. 加载态 加载中

⏳ 扫描 skill 目录时
首次加载或刷新时，从配置的来源目录扫描 skill，展示加载进度。
正在扫描 skill 目录...

7. Toast 反馈汇总

Toast 规则：
操作 类型 文案 时长
添加来源目录成功 success "已添加 ×× 目录，找到 × 个 skill" 2秒
移除来源目录 info "已移除 ×× 目录" 2秒
启用 skill 到应用 success "brainstorming 已添加到 Claude Code" 2秒
禁用 skill info "brainstorming 已从 Claude Code 移除" 2秒
保存项目配置 success "SkillDock 配置已保存" 2秒
扫描目录失败 error "无法访问 ×× 目录，请检查路径或权限" 5秒
复制配置到项目 success "配置已复制到 demo 项目" 2秒

Toast 展示规则

  • 位置：右下角，距离底部 20px，距离右侧 20px
  • 最大同时显示：3 条
  • 超过 3 条时，队列显示
  • 成功/信息：2 秒后自动消失
  • 错误：5 秒后自动消失
  • 点击 × 可手动关闭

8. 边界情况

8.1 长文本截断

Skill 名称截断
this-is-a-very-long-skill-name-that-should-be-truncated
max-width: 200px, text-overflow: ellipsis
描述文本截断
这是一个很长很长的 skill 描述文本，当它超过两行时应该显示省略号而不是完全展示出来，这样会影响界面布局。
2 行后省略，line-clamp: 2

8.2 大量数据滚动

└─ 超过 100 个 skill 时使用虚拟滚动
└─ 保持滚动流畅，不渲染不可见区域
└─ 侧边栏固定定位

8.3 目录路径过长

/Users/mac/Documents/Claude/skills/superpowers/very-deep-nested-category/subcategory/...
解决方案：路径使用 ~ 缩写，hover 显示完整路径

9. 交互行为规则表

触发 用户操作 系统行为 反馈
用户 点击"添加来源" 打开目录选择器对话框 Toast: "已添加 ×× 目录"
用户 点击 skill 开关 切换启用状态，保存到配置文件 Toast: "已添加/移除"
用户 点击侧边栏分类 切换右侧列表显示内容 无
用户 切换顶部应用下拉 加载对应应用的 skill 配置 无
用户 输入搜索关键词 实时过滤 skill 列表 无结果时显示空态
用户 点击 skill [详情] 打开 skill 详情弹窗 弹窗展示 SKILL.md 内容
用户 点击 [保存配置] 保存当前配置到配置文件 Toast: "配置已保存"
系统 启动时检测当前目录 自动加载对应项目配置 无
系统 扫描目录失败 记录失败目录，继续扫描其他 Toast: "部分目录加载失败"
异常 配置文件损坏 提示用户重置或手动修复 Toast: "配置文件异常"
