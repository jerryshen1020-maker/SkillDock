# SkillDock v1.2 需求总结

> **版本**: v1.2
> **日期**: 2026-03-04
> **状态**: 已确认

---

## 1. 背景

v1.1 完成了基础的 Skill 管理和项目级配置功能。在调研 cc-switch 和 Skills Hub 后，发现：
- Claude Code 的 `settings.json` 权限系统不支持 `Skills:*` 规则
- 可行的方案是通过文件系统管理（SSOT 架构）
- 现有方案仅支持应用级别控制，不支持项目级别

因此 v1.2 调整为纯全局应用级别的 Skill 管理，移除项目功能，并扩展多应用支持。

---

## 2. 目标

- 移除项目级别 Skill 控制功能
- 移除"全部项目"和"常用项目"功能
- 扩展支持多 AI 应用的 Skill 管理：Claude Code、Codex、OpenCode、Trae、Trae CN
- 支持本地目录来源和 Git 仓库来源
- 每个应用有独立的 Skill 列表界面

---

## 3. 功能范围

### 3.1 必须做

| 功能 | 说明 |
|------|------|
| 多应用管理 | 支持 Claude/Codex/OpenCode/Trae/Trae CN 五个应用 |
| 应用切换 | 左侧 hover 切换不同应用 |
| Skill 列表 | 展示当前应用的 Skills，支持删除 |
| 一键同步 | 每个应用界面有同步按钮，点击同步所有 Skills |
| 搜索 | 全局搜索 Skill（名称/描述/来源） |
| 来源管理 | 管理本地目录和 Git 仓库来源 |
| 添加本地目录 | 使用系统目录选择器添加来源 |
| 添加 Git 仓库 | 从 Git 克隆仓库作为来源 |
| Git 一键更新 | 批量更新所有已添加的 Git 仓库来源 |
| 设置面板 | 版本号显示、应用主题切换 |
| 同步机制 | 符号链接同步，失败时显示错误 |

### 3.2 不做

| 功能 | 原因 |
|------|------|
| 项目级别控制 | 调研证实 Skill 通过全局目录加载，无法实现项目隔离 |
| 项目管理 | 简化产品设计，专注核心功能 |
| 复制同步 | 失败时只显示错误，不自动回退复制 |
| 快捷操作 | 设置面板简化，只保留版本号和主题 |

---

## 4. 关键决策

| 决策项 | 选项 | 最终选择 |
|--------|------|----------|
| UI 架构 | 侧边栏应用列表 + 顶部 Tab 切换 | **侧边栏应用列表** |
| 应用选择 | Tab 选中 + radio 单选 | **纯 hover 选中**（无 radio） |
| 应用图标 | 无图标 / 统一图标 | **每个应用专属颜色 logo** |
| 来源入口 | 首页展示 / 独立页面 | **独立页面**（点击切换） |
| 来源 icon | ⚙️ / 🗂️ | **🗂️** |
| 添加来源 | 合并界面 / 独立界面 | **独立弹窗**（本地目录和 Git 分别弹窗） |
| 设置入口 | 左侧底部 / 右上角 | **左侧底部** |
| 设置内容 | 弹窗 / 独立页面 | **独立页面**（点击切换） |
| 同步方式 | 符号链接 + 复制 / 仅符号链接 | **仅符号链接**，失败时显示错误 |

## 5. UI 结构

### 5.1 首页布局

```
┌─────────────────────────────────────────────────────────────┐
│  🧠 SkillDock         [🔍 搜索 skills...]                  │
├──────────┬────────────────────────────────────────────────┤
│          │  Claude - Skills            共 12 个   [↻一键同步]│
│ [C]Claude│  ─────────────────────────────────────────     │
│ [C]Codex │                                                 │
│ [O]OpenCod│  ┌──────────┐  ┌──────────┐                  │
│ [T]Trae  │  │ Skill A  │  │ Skill B  │                  │
│ [T]TraeCN│  │   [删除] │  │   [删除] │                  │
│ ─────   │  └──────────┘  └──────────┘                  │
│ 🗂️ 来源   │                                                 │
│ ⚙️ 设置   │  ┌──────────┐  ┌──────────┐                  │
│          │  │ Skill C  │  │ Skill D  │                  │
│          │  │   [删除] │  │   [删除] │                  │
│          │  └──────────┘  └──────────┘                  │
└──────────┴────────────────────────────────────────────────┘
```

- 左侧移除"应用"文字标题
- 标题区域右侧增加"一键同步"按钮，点击同步当前应用的所有 Skills

### 5.2 页面与弹窗

**页面**（点击侧边栏切换）：

| 页面 | 触发 | 内容 |
|------|------|------|
| Skills 列表 | 点击应用名称 | 当前应用的 Skills 卡片列表 |
| 来源管理 | 点击左侧"来源" | 本地目录 + Git 仓库管理 |
| 设置 | 点击左侧"设置" | 版本号 + 应用主题 |

**弹窗**（点击触发）：

| 弹窗 | 触发 | 内容 |
|------|------|------|
| 添加目录 | 来源页面"+ 添加目录" | 目录选择器 |
| 添加仓库 | 来源页面"+ 添加仓库" | Git URL 输入 + 分支 |

## 6. 技术约束

### 6.1 应用 Skills 目录

| 应用 | skills 目录 (相对于 ~) |
|------|----------------------|
| Claude Code | `.claude/skills` |
| Codex | `.codex/skills` |
| OpenCode | `.config/opencode/skills` |
| Trae | `.trae/skills` |
| Trae CN | `.trae-cn/skills` |

### 6.2 同步策略

- **默认**：优先使用符号链接
- **失败**：显示错误 Toast，不自动回退复制
- **原因**：参考 Skills Hub 的处理方式，保持失败透明性

### 6.3 数据存储

- 应用级配置：UserDefaults
- 来源配置：本地 JSON 文件或 SQLite

## 7. 验收检查清单

### 7.1 功能验收

- [ ] 左侧应用列表显示 5 个应用（无"应用"标题）
- [ ] 应用列表通过 hover 切换
- [ ] 每个应用界面显示"一键同步"按钮
- [ ] 点击"一键同步"触发同步当前应用的所有 Skills
- [ ] 搜索框可过滤 Skills
- [ ] Skills 卡片显示名称、描述、删除按钮
- [ ] 点击"来源"切换到来源管理页面
- [ ] 可添加本地目录来源
- [ ] 可添加 Git 仓库来源
- [ ] 可一键更新所有 Git 仓库来源
- [ ] 可删除来源
- [ ] 点击"设置"切换到设置页面
- [ ] Git 同步失败显示错误

### 7.2 状态覆盖

- [ ] 正常态：有 Skills 展示
- [ ] 加载态：spinner + 提示文案

### 7.3 Toast 反馈

- [ ] 成功：2s 后消失
- [ ] 警告：3s 后消失
- [ ] 错误：5s 后消失
