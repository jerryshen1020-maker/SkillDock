# SkillDock v1.5 需求总结

> **版本**: v1.5
> **日期**: 2026-03-18
> **状态**: 设计探索完成，待开发实施
> **设计稿**: `01-设计稿.html` | `02-全状态设计参考.html`

## 1. 背景与目标

### 1.1 背景说明
SkillDock v1.3 已完成基础功能交付，支持 5 个 AI 工具（Claude Code、Codex、OpenCode、Trae、Trae CN）的技能管理。根据产品规划，需要：
- 扩展支持更多 AI 工具（WorkBuddy、CodeBuddy、Aion UI、Qoder）
- 优化技能浏览体验，提供"已安装"与"本地仓库"双视角
- 支持国际化，满足海外用户使用需求

### 1.2 目标
1. **应用扩展**：新增 4 个应用，完善 AI 工具生态覆盖
2. **分类浏览**：提供"已安装"和"本地仓库"两种视角，方便用户对比查看
3. **国际化**：支持中英双语，默认英文，可切换中文

## 3. 关键决策

### 3.1 分类 Tab 方案
**决策**：采用 Content Header 内嵌 Tab 方案

**理由**：
- Tab 位置在 title 下方、列表上方，视觉层次清晰
- 切换后保持 Tab 可见，方便用户随时切换
- 与现有布局结构兼容，改动最小

**替代方案**：
- 侧边栏入口（增加侧边栏层级，❌）
- 双栏对比（小屏幕拥挤，❌）

### 3.2 语言设置位置
**决策**：语言设置放在设置页面，不在 Title Bar

**理由**：
- 语言切换低频操作，不适合放在主界面
- 设置页面已有配置入口，符合用户心智
- Title Bar 保持简洁，只保留核心功能（搜索）

### 3.3 国际化实现方案
**决策**：使用 String Catalog (.xcstrings)

**理由**：
- Xcode 15+ 官方推荐方案
- 支持自动提取和 plurals、设备变体
- 便于后续扩展更多语言

**技术约束**：
- 所有文案需要支持中英双语
- 默认语言：英文
- 当前已支持中文，切换英文后需要界面适配

### 3.4 新增应用路径配置
**决策**：4 个新应用的 Skills 目录路径

| 应用 | 枚举值 | 路径 |
|------|--------|------|
| WorkBuddy | workBuddy | ~/.workbuddy/skills/ |
| CodeBuddy | codeBuddy | ~/.codebuddy/skills/ |
| Aion UI | aionUI | ~/.aionui-config/skills/ |
| Qoder | qoder | ~/.qoder/skills/ |

## 4. 技术约束

### 4.1 模型扩展
**AppTarget 枚举扩展**：
```swift
enum AppTarget: String, Codable, CaseIterable, Identifiable {
    // 现有 5 个应用
    case claudeCode, codex, openCode, trae, traeCN

    // v1.5 新增
    case workBuddy, codeBuddy, aionUI, qoder

    var defaultSkillsPath: String {
        switch self {
        case .workBuddy: return "~/.workbuddy/skills/"
        case .codeBuddy: return "~/.codebuddy/skills/"
        case .aionUI: return "~/.aionui-config/skills/"
        case .qoder: return "~/.qoder/skills/"
        // ...
        }
    }
}
```

### 4.2 分类视图模式
**新增枚举**：
```swift
enum SkillViewMode: String, Codable {
    case installedOnly   // 已安装
    case sourceRepository // 本地仓库
}
```

**AppConfig 扩展**：
```swift
struct AppConfig: Codable {
    // 现有字段
    var sources: [Source]
    var selectedApp: AppTarget
    // ...

    // v1.5 新增
    var skillViewMode: SkillViewMode
    var language: Language
}

enum Language: String, Codable {
    case english   // 默认
    case chinese
}
```

### 4.3 国际化框架
- 使用 String Catalog (.xcstrings)
- 所有文案使用 `LocalizedStringKey` 或 `NSLocalizedString`
- 语言切换后重新加载视图

### 4.4 UI 规范
严格遵循 `docs/specs/v1.3-ui-specs.md`：
- Design Tokens 不变
- 布局结构不变
- Tab 样式：灰色背景/蓝色激活态

## 5. 交互规则

### 5.1 分类 Tab 切换
| 用户操作 | 系统行为 | 反馈 |
|----------|----------|------|
| 点击"已安装" | 扫描目标目录，显示已安装技能 | loading → 列表 |
| 点击"本地仓库" | 扫描来源目录，显示所有技能 | loading → 列表 |

### 5.2 技能安装
| 用户操作 | 系统行为 | 反馈 |
|----------|----------|------|
| 在本地仓库点击"安装" | 创建符号链接到目标目录 | Toast "已安装 xxx" |
| 已安装技能显示"安装"按钮 | 禁用，显示"已安装" | - |

### 5.3 语言切换
| 用户操作 | 系统行为 | 反馈 |
|----------|----------|------|
| 切换语言选项 | 更新配置，重新加载文案 | Toast "语言已切换至 English" |

## 8. 风险与依赖

### 8.1 技术风险
| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 新应用路径不存在 | 扫描失败 | 提供默认路径，支持用户自定义 |

| 符号链接权限问题 | 安装失败 | 提供错误提示和重试入口 |

- 无新增外部依赖
- 遵循现有 v1.3 架构和规范
