# SkillDock v1.3 需求总结

**版本**: v1.3
**日期**: 2026-03-05
**状态**: 设计完成，待开发

---

## 1. 背景

v1.2 上线后，用户反馈以下体验问题：
1. Git 操作会阻塞 UI
2. 技能列表展示逻辑与心智不符（显示所有来源而非已安装）
3. 开关组件语义错误（关闭不会真正禁用 skill）
4. 交互细节需优化

---

## 2. 目标

- 技能列表只显示当前应用已全局安装的 skill
- Skill 操作改为「移除」和「清空全部」，真正删除符号链接
- Git 操作静默执行，不阻塞 UI
- 优化交互细节（hover 改 click，动画流畅）

---

## 3. 功能范围

### 3.1 技能列表重构

| 需求 | 描述 |
|------|------|
| P1-1 | 每个应用只显示已安装的 skill（从目标目录读取） |
| P1-2 | Skill 移除按钮：点击后删除当前应用的符号链接 |
| P1-3 | 清空全部按钮：二次确认后删除目标目录下所有内容（含历史遗留） |
| P1-4 | 首次安装/清空后显示空状态，引导同步 |
| P1-5 | 添加来源后需手动同步，不自动展示 |

### 3.2 Git 操作优化

| 需求 | 描述 |
|------|------|
| P2-1 | Git 克隆/更新在后台静默执行 |
| P2-2 | 在来源列表显示下载进度 |
| P2-3 | 不弹窗等待，不阻塞其他操作 |

### 3.3 限制调整

| 需求 | 描述 |
|------|------|
| P3-1 | 移除来源目录 10 个限制 |

### 3.4 交互优化

| 需求 | 描述 |
|------|------|
| P4-1 | 侧边栏应用切换：hover 改为 click |
| P4-2 | 修复转场动画卡顿 |

---

## 4. 不做范围

- Skill 开关功能（改为移除按钮）
- 项目维度管理（保持纯应用级别）
- 自动同步能力（保持手动触发）

---

## 5. 关键决策

| # | 决策 | 理由 |
|---|------|------|
| 1 | Skill 移除改为删除符号链接 | 调研确认 `settings.json` 权限对 Skill 无效，必须从文件系统层面控制 |
| 2 | 从目标目录读取已安装 skill | 与 cc-switch/Skills Hub 方案一致，确保展示与实际一致 |
| 3 | 清空全部包含历史遗留 | 确保用户能清理非 SkillDock 管理的旧 skill |
| 4 | Git 静默后台执行 | 避免大仓库阻塞主线程，提升体验 |

## 6. 技术约束

### 6.1 各应用 skills 目录

| 应用 | 目录 |
|------|------|
| Claude Code | `~/.claude/skills` |
| Codex | `~/.codex/skills` |
| OpenCode | `~/.config/opencode/skills` |
| Trae | `~/.trae/skills` |
| Trae CN | `~/.trae-cn/skills` |

### 6.2 同步策略

- 符号链接优先
- 失败时显示错误（不回退复制）
- 清理历史托管链接

---

## 7. UI 变更点

### 7.1 技能卡片

- 移除 Toggle 开关
- 右上角改为「移除」按钮
- 点击后二次确认，删除符号链接

### 7.2 顶部区域

- 新增「清空全部」按钮（黄色，与一键同步并列）
- 空状态不显示「清空全部」

### 7.3 空状态

- 显示「暂无已安装的 Skill」
- 只有一个「一键同步」按钮引导

### 7.4 来源管理

- Git 操作显示进度指示
- 后台静默执行
