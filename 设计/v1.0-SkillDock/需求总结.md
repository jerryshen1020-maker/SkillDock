# SkillDock 需求总结

## 项目概述

**项目名称**: SkillDock
**版本**: v1.0
**类型**: macOS 独立应用
**设计风格**: 标准 SwiftUI

## 背景与目标

### 痛点
- skill 分散在多个目录（~/.claude/skills/, /Users/mac/Documents/Claude/skills/common, superpowers, mini-agent 等）
- 不同项目需要不同的 skill，但无法按项目隔离
- 全局 skill 太多，不便于管理和查找
- 不同 AI 工具（Claude Code, Codex, Trae, Trae CN）调用方式不同

### 目标
- 统一管理所有 skill
- 支持按应用分配 skill（Claude Code, Codex, Trae, Trae CN）
- 支持按项目分配 skill，实现目录隔离
- 提供可视化界面预览和配置

## 功能范围

### 做什么
1. **Skills仓库**
   - 汇总多个来源目录的 skill
   - 支持配置来源目录（添加/删除）
   - 展示所有 skill 及其能力描述

2. **应用配置**
   - 为不同应用（Claude Code, Codex, Trae, Trae CN）独立配置可用 skill
   - Trae/Trae CN 需手动在软件中导入（⚠️ 可能不支持外部接入）

3. **项目配置**
   - 每个项目目录独立配置可用的 skill
   - 支持常用项目收藏
   - 自动检测当前工作目录

4. **交互界面**
   - macOS 原生客户端
   - 侧边栏分类（Skills仓库、常用项目、全部项目）
   - 右侧顶部应用切换下拉菜单

### 不做什么
- 暂时不做 skill 的增删改（只做展示、选择和配置）
- 暂时不做 skill 的导入/导出功能
- Trae/Trae CN 的自动配置（仅支持手动导入标注）

## 关键决策

1. **设计风格**: 标准 SwiftUI（稳定优先，兼容性好）
2. **载体**: 独立 macOS 应用
3. **项目切换**: 支持常用收藏 + 全部列表
4. **应用配置**: 每个应用独立配置，Trae 标注待调研
