触发 用户操作 系统行为 反馈
用户 点击「一键同步」 扫描所有来源目录，比对新增/删除/变更 Toast: 同步中 → 同步完成/失败
用户 点击「添加来源」 弹出系统目录选择器 选择后验证目录有效性

用户 点击侧边栏入口 切换内容区视图 高亮当前选中项

5. 技术约束说明

5.1 Skill 开关生效机制

要真正限制 Skill 被 Claude 调用，需要修改项目级 <project>/.claude/settings.json 中的 permissions 字段。

配置结构：

// 项目目录: <project>/.claude/settings.json
{
  "permissions": {
    "allow": ["Skill(brainstorming)", "Skill(design-exploration)"],
    "deny": ["Skill(browser-use)"]
  }
}

项目隔离原理： 即使所有 skill 安装在全局目录 ~/.claude/skills/，通过项目级 permissions 配置，可以实现不同项目使用不同 skill。

5.2 递归扫描

扫描时自动递归遍历子目录，识别包含 SKILL.md 的文件夹作为有效 Skill。

5.3 默认全局目录

首次启动时自动添加 ~/.claude/skills/ 作为默认来源。
