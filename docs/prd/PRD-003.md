# 产品需求文档：SkillDock - V1.2

## 1. 背景 (Background)

### 1.1 产品定位

| 项目 | 内容 |
|------|------|
| 名称 | SkillDock |
| 版本 | v1.2 |
| 类型 | macOS 原生应用 (SwiftUI) |
| 兼容性 | macOS 12.0+ |

### 1.2 V1.1 调研发现

在调研 cc-switch 和 Skills Hub 后，发现：
- **Claude Code 不支持 Skill 权限控制**：`settings.json` 中的 `Skills:*` 规则无效
- **可行的替代方案**：通过文件系统管理（SSOT 架构）
- **现有方案局限性**：仅支持应用级别控制，不支持项目级别

### 1.3 需求变更

| 变更项 | 变更前 | 变更后 |
|--------|--------|--------|
| 控制粒度 | 项目级 + 应用级 | 仅应用级别 |
| 项目功能 | 全部项目、常用项目 | 移除，改为纯全局控制 |
| 应用支持 | 仅 Claude Code | 扩展到 5 个 AI 工具 |

## 2. 目标 (Goals)

### 2.1 产品目标

- **移除项目功能**：简化产品设计，专注核心功能
- **多应用支持**：支持 Claude Code、Codex、OpenCode、Trae、Trae CN 五个应用
- **统一管理**：通过应用独立界面管理各自的 Skills
- **来源扩展**：支持本地目录和 Git 仓库两种来源

### 2.2 设计原则

- 简洁优先：移除项目维度，简化导航
- 渐进增强：在 v1.1 基础上扩展多应用支持
- 用户体验：保持一致的 UI 风格和交互体验

## 3. 功能范围 (Feature Scope)

### 3.1 本次包含 (In Scope)

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 多应用管理 | P0 | 支持 5 个 AI 应用的 Skill 管理 |
| 应用切换 | P0 | 左侧 hover 切换不同应用 |
| Skill 列表 | P0 | 展示当前应用的 Skills，支持删除 |
| 一键同步 | P0 | 每个应用界面有同步按钮 |
| 搜索 | P1 | 全局搜索 Skill（名称/描述/来源） |
| 来源管理 | P0 | 管理本地目录和 Git 仓库来源 |
| 添加本地目录 | P0 | 使用系统目录选择器添加 |
| 添加 Git 仓库 | P0 | 从 Git 克隆仓库作为来源 |
| Git 一键更新 | P1 | 批量更新所有 Git 仓库 |
| 设置 | P1 | 版本号显示、应用主题切换 |
| 同步机制 | P0 | 符号链接同步，失败时显示错误 |

### 3.2 本次不包含 (Out of Scope)

- 项目级别 Skill 控制
- 项目管理（全部项目、常用项目）
- 复制同步（失败时只显示错误）
- Skill 的启用/禁用开关（改为直接删除）

### 4.2 同步策略

- **默认**：优先使用符号链接
- **失败**：显示错误 Toast，不自动回退复制
- **原因**：参考 Skills Hub 的处理方式，保持失败透明性

### 4.3 数据存储

- 应用级配置：UserDefaults
- 来源配置：本地 JSON 文件或 SQLite

## 6. 关键决策 (Key Decisions)

1. **UI 架构**：侧边栏应用列表，通过 hover 切换
2. **应用选择**：纯 hover 选中（无 radio），每个应用有专属颜色 logo
3. **来源入口**：独立页面（点击切换）
4. **添加来源**：独立弹窗（本地目录和 Git 分别弹窗）
5. **Git 一键更新**：在来源管理页面的 Git 仓库区域提供批量更新按钮
6. **同步方式**：仅符号链接，失败时显示错误
7. **Skill 删除**：直接删除，不保留启用/禁用状态

### 8.2 加载态

| 场景 | 图标 | 提示 |
|------|------|------|

|------|------|------|

## 9. 交互反馈 (Toast 规则)

### 9.1 Toast 规范

| 类型 | 时长 | 位置 | 同时最多 |
|------|------|------|----------|
| 成功 | 2 秒 | 右下角 | 3 条 |
| 警告 | 3 秒 | 右下角 | 2 条 |
| 错误 | 5 秒 | 右下角 | 1 条 |

### 9.2 Toast 场景

| 场景 | 类型 | 文案 |
|------|------|------|
| 添加来源成功 | 成功 | ✓ 已添加来源目录 |
| 删除来源 | 成功 | ✓ 已删除来源 |
| Git 仓库添加 | 成功 | ✓ 正在克隆仓库... |
| Git 一键更新 | 成功 | ✓ 已更新 3 个仓库 |
| Git 同步失败 | 错误 | ✗ 同步失败：{原因} |
| 删除 Skill | 成功 | ✓ 已删除 Skill |
| 目录不存在 | 警告 | ⚠️ 目录不存在，已标记 |

## 10. 交互行为规则表

| 触发 | 用户操作 | 系统行为 | 反馈 |
|------|----------|----------|------|
| 用户 | hover 应用列表 | 切换到对应应用的 Skills 列表 | 高亮当前应用 |
| 用户 | 点击"一键同步" | 同步当前应用的所有 Skills | Toast: 同步中 → 完成/失败 |
| 用户 | 输入搜索关键词 | 实时过滤 Skills（名称/描述/来源） | 无结果时显示空态 |
| 用户 | 点击 Skill"删除" | 从应用目录删除该 Skill | Toast: 已删除 Skill |
| 用户 | 点击左侧"来源" | 切换到来源管理页面 | - |
| 用户 | 点击"+ 添加目录" | 打开目录选择器，选择后添加到来源 | Toast: 已添加来源目录 |
| 用户 | 点击"+ 添加仓库" | 从 Git 克隆仓库 | Toast: 正在克隆... |
| 用户 | 点击"↻ 一键更新" | 批量更新所有 Git 仓库 | Toast: 更新中 → 完成/失败 |
| 用户 | 点击来源"删除" | 删除该来源配置 | Toast: 已删除来源 |
| 用户 | 点击左侧"设置" | 切换到设置页面 | - |
| 系统 | 启动时 | 加载配置，扫描所有来源目录 | 显示加载态 → 正常态/错误态 |
| 系统 | 检测到目录不存在 | 标记该来源为不可用 | Toast 警告 + 错误态 UI |
| 系统 | Git 同步失败 | 显示错误状态 | Toast Error + 错误态 UI |

## 11. 术语表 (Glossary)

| 术语 | 定义 |
|------|------|
| Skill | AI 工具的技能包，包含 SKILL.md 描述文件 |

| 符号链接 | 同步方式的一种，快速且不占用额外空间 |
