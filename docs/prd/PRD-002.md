# 产品需求文档：SkillDock - V1.1

> 归档说明（2026-03-05）：本文件为 v1.1 历史需求，含阶段性“项目级隔离”设计，不代表当前实现。当前实现请以 `docs/prd/PRD-003.md` 与 `docs/logs/PROJECT_LOG.md` 为准。

## 1. 背景 (Background)

### 1.1 产品定位

| 项目 | 内容 |
|------|------|
| 名称 | SkillDock |
| 版本 | v1.1 |
| 类型 | macOS 原生应用 (SwiftUI) |
| 兼容性 | macOS 12.0+ |

### 1.2 V1.0 用户反馈痛点

1. **路径输入体验差**：需要手动输入路径，容易出错
2. **扫描能力不足**：不能递归扫描子文件夹内的 skill
3. **缺少默认配置**：首次使用需要手动添加全局目录
4. **UI 与设计稿差异大**：侧边栏结构复杂，不符合现代审美
5. **Skill 开关无效**：本地配置未同步到 Claude 真正生效的配置

## 2. 目标 (Goals)

### 2.1 产品目标

- 提升易用性：通过系统目录选择器和默认配置，降低首次使用门槛
- 增强功能性：支持递归扫描，发现更多 skill
- 完善 UI：对齐设计稿，提供现代化视觉体验
- 实现真正生效：Skill 开关同步到 Claude 配置

### 2.2 设计原则

- 简洁优先：侧边栏简化为入口式导航
- 渐进增强：保留 v1.0 核心功能，叠加新能力
- 兼容性：确保现有数据和配置平滑迁移

## 4. 技术方案 (Technical Solution)

### 4.1 目录选择器 (NSOpenPanel)

```swift
// macOS 原生目录选择器
func selectDirectory() -> String? {
    let panel = NSOpenPanel()
    panel.canChooseFiles = false
    panel.canChooseDirectories = true
    panel.allowsMultipleSelection = false
    let response = panel.runModal()
    return response == .OK ? panel.url?.path : nil
}
```

### 4.2 递归扫描

```swift
// FileManager 递归遍历
func scanDirectoryRecursively(_ path: String) -> [URL] {
    let fileManager = FileManager.default
    guard let enumerator = fileManager.enumerator(at: URL(fileURLWithPath: path),
                                                   includingPropertiesForKeys: nil) else {
        return []
    }
    var skillDirs: [URL] = []
    for case let url as URL in enumerator {
        let skillMD = url.appendingPathComponent("SKILL.md")
        if fileManager.fileExists(atPath: skillMD.path) {
            skillDirs.append(url)
        }
    }
    return skillDirs
}
```

### 4.3 默认全局目录

应用启动时检测并导入：
- 路径：`~/.claude/skills/`
- 标记为内置来源，不可删除
- 仅首次启动时自动添加

### 4.4 Skill 开关生效机制

**Claude CLI 项目级配置结构：**
```json
// <project>/.claude/settings.json
{
  "permissions": {
    "allow": ["Skill(brainstorming)", "Skill(design-exploration)"],
    "deny": ["Skill(browser-use)"]
  }
}
```

**优先级规则：**
- Project > User
- 项目级配置覆盖全局设置
- 即使所有 skill 安装在全局目录，也可通过项目级配置实现隔离

**写入逻辑：**
1. 用户切换项目时，读取该项目的 .claude/settings.json
2. 用户修改开关时，同时更新本地 .skills-config.json 和 .claude/settings.json
3. 检测并创建 .claude 目录（如不存在）

### 4.5 项目隔离原理

即使所有 skill 安装在全局目录 `~/.claude/skills/`，通过项目级 permissions 配置：
- 项目 A：允许 brainstorming, design-exploration
- 项目 B：允许 prd-doc-writer, writing-assistant

## 6. 关键决策 (Key Decisions)

1. **入口式侧边栏**：简化侧边栏，点击入口切换内容区
2. **来源筛选整合**：在 Skills 仓库页面顶部添加来源筛选器
3. **系统目录选择器**：使用 NSOpenPanel 替代手动输入
4. **Skill 开关生效**：修改项目级 `.claude/settings.json` 的 permissions 配置
5. **项目隔离**：通过 Project 级配置实现不同项目使用不同 skill

## 9. 交互反馈 (Toast 规则)

### 9.1 Toast 规范

| 类型 | 时长 | 位置 | 同时最多 |
|------|------|------|----------|
| 成功 | 2 秒 | 右下角 | 3 条 |
| 警告 | 3 秒 | 右下角 | 2 条 |
| 错误 | 5 秒 | 右下角 | 1 条 |

### 9.2 Toast 场景

| 场景 | 类型 | 文案 |
|------|------|------|
| 一键同步完成 | 成功 | ✓ 同步完成，新增 3 个 Skill |
| 添加来源成功 | 成功 | ✓ 已添加来源 |
| 移除来源 | 成功 | ✓ 来源已移除 |
| Skill 启用/禁用 | 成功 | ✓ brainstorming 已启用 |
| 添加项目 | 成功 | ✓ 项目已添加 |
| 目录不存在 | 警告 | ⚠️ 目录不存在，请检查路径 |
| 权限不足 | 错误 | ✗ 无访问权限 |
| 同步失败 | 错误 | ✗ 同步失败，请重试 |
| 项目已存在 | 警告 | ⚠️ 该项目已存在 |

## 10. 交互行为规则表

| 触发 | 用户操作 | 系统行为 | 反馈 |
|------|----------|----------|------|
| 用户 | 点击「一键同步」 | 扫描所有来源目录，比对新增/删除/变更 | Toast: 同步中 → 同步完成/失败 |
| 用户 | 点击「添加来源」 | 弹出系统目录选择器 | 选择后验证目录有效性 |
| 用户 | 点击 Skill Toggle | 更新配置状态，写入 .claude/settings.json | Toast: 已启用/已禁用 |
| 用户 | 输入搜索关键词 | 实时过滤 Skills 列表（名称/描述） | 无结果时显示空态 |
| 用户 | 点击来源筛选标签 | 过滤指定来源的 Skills | 无结果时显示空态 |
| 用户 | 点击侧边栏入口 | 切换内容区视图 | 高亮当前选中项 |
| 用户 | 点击「添加项目」 | 弹出目录选择器，选择项目目录 | 验证后添加并扫描 |
| 系统 | 启动时 | 加载配置，扫描来源目录 | 显示加载态 → 正常态/错误态 |
| 系统 | 检测到目录不存在 | 标记该来源为不可用 | Toast 警告 + UI 提示 |
| 系统 | 切换项目 | 加载项目级配置，更新 Skills 状态 | Skills 列表反映项目配置 |
