# SkillDock v1.2 全量自动化回归测试 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 建立覆盖 PRD、设计稿、技术方案、可用性的全量自动化回归体系，并作为 CI 强门禁阻断回归。

**Architecture:** 采用四层回归架构：单元/集成测试保障业务正确性，XCUITest 保障关键用户流程，视觉回归保障像素级设计一致性，Design Contract 测试保障关键尺寸与颜色约束。所有 PRD 验收项必须映射到自动化测试 ID，任何未覆盖项禁止发布。

**Tech Stack:** Swift/XCTest/XCUITest、xcodebuild、Python3（截图 diff 脚本）、CI（GitHub Actions 或同等流水线）

## 1. 自动化测试分层与发布门禁

### Layer A：单元测试（业务规则）

- 覆盖对象：`Models/Services/ViewModels`
- 目标：每个 PRD 验收点至少 1 条可追踪单测
- 必测主题：
  - 多应用切换与持久化
  - 来源新增/删除/重试/Git 更新
  - 同步冲突与导出失败分级
  - 设置主题模式持久化

### Layer B：集成测试（本地文件系统与 Git 行为）

- 覆盖对象：符号链接写入、目录存在性、权限失败、Git clone/pull 失败
- 目标：失败路径必须具备稳定断言，不允许只测 happy path

### Layer C：UI 自动化（XCUITest）

- 覆盖对象：关键用户旅程
- 目标：从用户视角验证“可操作 + 可反馈 + 可恢复”
- 必测旅程：
  - 应用切换 → 搜索过滤 → 一键同步
  - 来源页添加本地目录/添加 Git/批量更新/删除
  - 设置页主题切换（system/light/dark）
  - 错误态展示与重试

### Layer D：视觉回归（像素级）

- 覆盖对象：主界面、来源页、设置页、空态/错误态/加载态/交互态
- 目标：100% 还原设计稿（允许仅对动态区域做 mask）
- 判定：默认零容差，超阈值直接 fail CI

### CI Gate（全部通过才可合并）

1. Build Gate：`xcodebuild -project SkillDock.xcodeproj -scheme SkillDock build`
2. Unit+Integration Gate：`xcodebuild test -project SkillDock.xcodeproj -scheme SkillDock`
3. UI Journey Gate：`xcodebuild test -project SkillDock.xcodeproj -scheme SkillDockUITests`
4. Visual Gate：`python3 tools/visual-regression/compare.py`
5. Coverage Mapping Gate：`python3 tools/testing/validate_prd_mapping.py`

```text
SkillDockTests/

SkillDockUITests/
  Journeys/

    SyncErrorJourneyUITests.swift
  States/
    EmptyLoadingErrorUITests.swift

    VisualSnapshotCaptureUITests.swift

tests-artifacts/
  baselines/
    v1.2/
      skills/
      source/
      settings/
      states/
  snapshots/
  diff/

tools/
  visual-regression/
    compare.py
    masks/
  testing/
    validate_prd_mapping.py
    prd_traceability.yml
```

### 命名规范

- 测试类：`<Feature><Layer>Tests`，如 `SourceManagementJourneyUITests`
- 测试方法：`test_<场景>_<条件>_<预期结果>`
- 用例 ID：`AC-7.3-004` / `DESIGN-012` / `STATE-006`
- 基线图：`<page>__<state>__<theme>__<size>.png`
- 失败 diff：`<baseline>__diff.png`

## 4. 可执行任务清单（按 TDD 执行）

### Task 1：建立 UI 测试 Target 与基础设施

**Files:**
- Modify: `project.yml`
- Create: `SkillDockUITests/Journeys/AppSwitchJourneyUITests.swift`
- Create: `SkillDockUITests/States/EmptyLoadingErrorUITests.swift`

**Step 1: 写首个失败 UI 测试**
- 目标：验证左侧 5 应用项存在且可切换

**Step 2: 运行测试确认失败**
- Run: `xcodegen generate && xcodebuild test -project SkillDock.xcodeproj -scheme SkillDockUITests -only-testing:SkillDockUITests/AppSwitchJourneyUITests`

**Step 3: 补齐最小可测标识**
- 为关键控件添加稳定可访问标识（如 app tab、sync button、source entry）

**Step 4: 再跑测试至通过**
- 同上命令，预期 PASS
